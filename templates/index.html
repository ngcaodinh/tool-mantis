Xem mô tả chi tiết tất cả các trường (Copy đưa cho AI sinh JSON)
1. Trường bắt buộc:
  • summary (String): Tiêu đề lỗi.
  • description (String): Mô tả chi tiết lỗi.
2. Trường tùy chọn:
  • category_id (Int/String): ID phân loại. (Tự chọn nếu đã chọn dropdown trên).
  • project_id (Int/String): ID dự án (Mặc định 0).
  • handler_id (Int/String): ID thành viên xử lý (Mặc định 0).
  • severity (String/Int): Độ nghiêm trọng ("feature", "trivial", "text", "tweak", "minor", "major", "crash", "block").
  • priority (String/Int): Độ ưu tiên ("none", "low", "normal", "high", "urgent", "immediate").
  • reproducibility (String/Int): Khả năng tái hiện ("always", "sometimes", "random", "have not tried", "unable to duplicate", "N/A").
  • eta (String/Int): Dự kiến hoàn thành ("none", "< 1 day", "2-3 days", "< 1 week", "< 1 month", "> 1 month").
  • platform / os / os_build / version / build (String): Thiết bị, hệ điều hành, build, phiên bản...
  • due_date (String): Hạn xử lý, định dạng "YYYY-MM-DD".
  • status (String/Int): Trạng thái lỗi ("new", "feedback", "acknowledged", "confirmed", "assigned", "resolved", "closed").
  • resolution (String/Int): Hướng xử lý ("open", "fixed", "reopened", "unable to reproduce", "not fixable", "duplicate", "no change required", "suspended", "won't fix").
  • view_state (String/Int): Chế độ hiển thị ("public", "private").
  • steps_to_reproduce (String): Các bước tái hiện.
  • additional_info (String): Thông tin thêm.
  • tag_string (String): Danh sách tags (phân cách bằng dấu phẩy).
  • custom_field_<id> (String): Custom Field (ví dụ: custom_field_1).
